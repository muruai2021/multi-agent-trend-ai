# Topic Template — 选题生成模板

> 本文件定义 multi-agent-trend-ai 的选题生成 prompt 模板与组合规则。
> 最后更新: 2026-06-19

---

## 1. 选题生成主 Prompt

```markdown
# Role
你是一位资深的 AI 公众号 / 视频号内容策略师，擅长把"技术热度"二次翻译成"创作者可落地的选题"。

# Task
基于以下【四梯队关键词报告】和【用户背景】，生成 2-3 条高命中选题组合。

# Inputs
- 报告日期: {report_date}
- 用户背景: {user_background}
- 目标平台: {target_platform}（公众号 / 视频号 / 两者）
- S 梯队词: {tier_s_keywords}
- A 梯队词: {tier_a_keywords}
- B 梯队词: {tier_b_keywords}
- C 梯队词: {tier_c_keywords}

# Constraints
1. 每条选题必须含 1 个 S 或 A 梯队词 + 1 个应用场景词
2. 标题长度: 公众号 15-25 字，视频号 10-20 字
3. 必须给出预估热度（基于所选词的 heat_score 加权）和发布建议
4. 至少包含 1 条"反共识"选题（避免与已有爆款雷同）

# Output Format (JSON)
{
  "topics": [
    {
      "rank": 1,
      "title_zh": "...",
      "title_en": "...",
      "keyword_combo": ["S词", "应用词"],
      "platform": "公众号 | 视频号 | 两者",
      "format": "深度长文 | 短视频 | 图文 | 案例拆解",
      "estimated_heat": 0-100,
      "rationale": "为什么这个组合能爆（1-2 句话）",
      "outline": ["章节1", "章节2", "章节3"]
    }
  ],
  "anti_consensus_topic": {
    "title_zh": "...",
    "rationale": "逆向选题的角度说明"
  }
}
```

## 2. 选题组合规则

### 2.1 优先级矩阵

| 主词梯队 | 配词推荐 | 平台推荐 | 形态 |
|---------|---------|---------|------|
| S 词 | B 词（应用场景） | 公众号 + 视频号 | 深度长文 + 短视频 |
| A 词 | B 词（应用场景） | 公众号 | 案例拆解 |
| B 词 | C 词（新兴） | 视频号 | 短视频 |
| C 词 | - | 视频号 | 30s 实验性内容 |

### 2.2 组合公式

```
高命中选题 = 1 × 高热度词 (S/A) + 1 × 应用场景词 (B) + 1 × 形态标签
```

应用场景词推荐库：
- 电商客服 / 内容运营 / 代码开发 / 数据分析 / 教育辅导 / 医疗咨询 / 财务管理

形态标签推荐：
- 实战教程 / 案例拆解 / 趋势解读 / 工具测评 / 行业访谈

## 3. 标准选题模板（5 种）

### 模板 1: 实战教程型
```
【{S 词}】30 天实战：{应用场景}完整流水线拆解
```

### 模板 2: 案例拆解型
```
从 0 到 1：用 {S 词} 改造 {应用场景}，{可量化收益}
```

### 模板 3: 趋势解读型
```
{S 词} 2026 上半年观察：{3 个关键变化}
```

### 模板 4: 工具测评型
```
{A 词} vs {A 词}：{应用场景}场景下谁更强？
```

### 模板 5: 反共识型
```
为什么 {S 词} 在 {垂直领域} 行不通？— 3 个真实踩坑案例
```

## 5. 用户背景适配

### 5.1 开发者背景
- 偏好：技术深度、代码示例、架构图
- 推荐模板: 模板 1, 模板 2
- 平台权重: 公众号 70% / 视频号 30%

### 5.2 写作者背景
- 偏好：叙事化、案例化、人物故事
- 推荐模板: 模板 2, 模板 3, 模板 5
- 平台权重: 公众号 60% / 视频号 40%

### 5.3 运营 / PM 背景
- 偏好：ROI、落地路径、工具对比
- 推荐模板: 模板 3, 模板 4
- 平台权重: 公众号 50% / 视频号 50%

### 5.4 通用（默认）
- 偏好：平衡、可读性
- 推荐模板: 模板 1, 模板 3
- 平台权重: 公众号 50% / 视频号 50%

---

## 6. 选题质量检查

每条生成的选题必须通过以下检查：

- [ ] 至少含 1 个 S 或 A 词
- [ ] 标题符合平台字数要求
- [ ] 有可量化的收益或案例
- [ ] 与最近 4 周历史选题去重（避免重复）
- [ ] 反共识选题必须有逆向角度说明
- [ ] 预估热度 ≥ 60（低于此值视为低命中）

未通过检查的选题自动重新生成，最多 3 次。

---

## 7. 选题保存

生成的选题写入 `outputs/topic-suggestions-{YYYYMMDD}.md`，包含：
- 选题清单
- 数据依据（哪些词、什么热度）
- 发布建议（最佳发布时间、引流路径）
- 复盘挂钩（1 周后回看实际效果）
